# ChatBI 测试样例

基于当前数据模型：
- `order_master`
- `order_detail`
- `user_info`
- `product_info`
- `store_info`
- `refund_master`
- `refund_detail`

数据特征：
- 订单主表约 `12万`
- 订单明细约 `26万+`
- 用户约 `4万`
- 覆盖品牌、品类、渠道、门店、大区、退款等主题

建议按以下顺序回归。

## 1. 基础冒烟

1. `统计订单总数`
2. `统计近30天销售金额`
3. `统计近30天订单数、销售金额、客单价`
4. `统计近30天退款金额和退款单数`

期望：
- 能直接返回结果
- 指标定义、指标描述正常
- SQL 执行成功

## 2. 商品分析

1. `按品牌统计近30天销量和销售金额，按销售金额降序`
2. `按品类统计近30天销售金额占比`
3. `统计特仑苏近30天销售金额、销量和退款金额`
4. `按产品名称统计近30天销量前20`
5. `按渠道统计纯甄近30天销售金额，并用柱图展示`

期望：
- 会优先走 `order_detail + product_info`
- 商品粒度金额使用明细金额，不重复汇总主表金额
- 图表按钮可切换

## 3. 区域与门店分析

1. `按销售大区统计近30天销售金额和订单数，按销售金额降序`
2. `统计河南门店近30天销售金额排名前100`
3. `按门店名称统计近30天退款金额前50`
4. `按组织架构统计近30天销售金额和退款金额`
5. `按城市层级统计近30天销售金额，并用饼图展示`

期望：
- 会优先联查 `store_info`
- 排名类问题若指标明确，直接返回结果

## 4. 用户分析

1. `按会员等级统计近30天销售金额和订单数`
2. `按性别统计近30天销售金额`
3. `按年龄段统计近30天订单数和销售金额`
4. `统计近30天新用户销售金额`
5. `按注册渠道统计近30天销售金额，并用柱图展示`

期望：
- 会优先联查 `user_info`
- 维度名称返回业务名称，不返回库字段名

## 5. 退款分析

1. `统计近30天退款金额、退款件数和退款单数`
2. `按退款原因统计近30天退款金额`
3. `按品牌统计近30天退款金额和退款件数`
4. `按门店统计近30天退款金额前20`
5. `按销售大区统计近30天退款金额，并用柱图和饼图展示`

期望：
- 会优先使用 `refund_master / refund_detail`
- 退款分析可与商品、门店维度联查

## 6. 连续对话与澄清

场景 A：
1. `查询订单排名前100的数据`
2. `按销售金额`

期望：
- 第一句先追问
- 第二句补全口径后继续查询

场景 B：
1. `统计河南地区的销售金额排名前100`
2. `把江苏的也拉出来`

期望：
- 第二句保留前文统计口径
- 自动扩成多地区查询

场景 C：
1. `按品牌统计近30天销量和销售金额`
2. `只看特仑苏和纯甄`
3. `再把时间改成近15天`

期望：
- 能继承前文维度和指标
- 时间范围组件或自然语言追问都能继续工作

## 7. 图表展示

1. `按品牌统计近30天销售金额`
2. 点击 `柱图`
3. 点击 `饼图`
4. 点击 `列表`

期望：
- 三种视图可切换
- 如果结果不适合图表，按钮应置灰

## 8. 导出能力

建议先执行：
- `按品牌统计近30天销量和销售金额，按销售金额降序`

然后验证：
1. 点击 `下载明细数据`
2. 点击 `下载图表Word`
3. 点击 `生成商业报告`

期望：
- 浏览器弹出保存位置
- CSV 可打开
- Word 文档可正常打开
- 图表快照已嵌入

## 9. 模板能力

1. 点击 `下载模板样例`
2. 在模板下拉框中选择：
   - `默认管理层商业分析报告模板`
   - `中国通用商业分析报告模板`
3. 上传一个你自己的 `.docx` 样例模板
4. 再生成商业报告

期望：
- 模板样例可下载
- 上传后模板出现在下拉框
- 报告生成时使用所选模板的样式
